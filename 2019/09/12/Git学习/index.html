Git学习

前端 Git 2019/09/12

一、Git基础

1.初次运行 Git 前的配置

git官方文档

  1. git config –global user.name “John Doe”
  2. git config –global user.email johndoe@example.com

缺省等同于local

  1. git config –local //local只对某个仓库有效
  2. git config –global //global对当前用户所有仓库有效
  3. git config –system //system对系统所有登录的用户有效

显示config的配置，加–list

  1. git config –list –local
  2. git config –list –global
  3. git config –list –system

2.给文件重命名的方法

  1. git reset –hard //取消暂存区
  2. git mv project new_project //给文件重命名

3.通过gitlog查看版本演变历史

  1. git log –oneline //产看日志的简洁方式
  2. git log –oneline –all //产看日志的简洁方式的所有分支
  3. git log -n2 //查看当前分支最近的两次log
  4. git log –all //查看所有分支的log
  5. git log –graph //看整个版本的演进历史

4.了解

  1. gitk ：通过图形界面工具查看版本历史 –all
  2. .git目录
  3. git checkout 分支名 //切换分支
  4. commit tree blob

5.分离头指针

  1. git checkout commitId
  2. git branch -av //查看分支

二、使用Git的常见场景

1.删除不需要的分支

  1. git branch -d
  2. git commit –amend //对最近提交的一次commit做变更

2.把本地仓库同步到GitHub

  1. git remote -v
  2. git remote add github git@github.com:lining0/git_learning.git
  3. git push github –all //本地仓库push到远端(报错)
  4. git fetch github master //把远端的拉下来
  5. git merge github/master //和远端的master做合并(报错)
  6. git merge –allow-unrelated-histories github/master
  7. git push github master //本地仓库push到远端

三、Git多人单分支集成协作时的常见场景

1.不同人修改了不同文件如何处理？

  1. git checkout -b feature/add_git_comments origin/feature/add_git_comments //基于远端的分支创建本地的分支，并切换分支
  2. git add -u //添加到暂存区
  3. git status //查看状态
  4. git commit -m’add_git_commands’ //记录
  5. git push //本地仓库push到远端
  6. git pull //拉取（包括两个步骤git fetch && git merge ）
  7. git fetch github //把远端的拉下来
  8. git merge origin/feature/add_git_comments //把远端的合并到本地

2.不同人修改了同文件的不同区域如何处理？

  1. git pull //远端拉取
  2. git merge

3.不同人修改了同文件的同一区域如何处理？

  1. git merge origin/feature/add_git_comments //把远端的合并到本地
  2. git pull
  3. git commit -am’Resolved conflict by hand with 4 git commands’
  4. git push

4.同时变更了文件名和文件内容如何处理？

  1. git pull
  2. git push

5.把同一文件改成了不同的文件名如何处理？

  1. git rm idnex.html
  2. git add idnex1.htm
  3. git rm idnex2.htm
  4. git commit -am’decide to mv index to index1’
  5. git push
